üéØ Understanding DSCP Markings:

DSCP (Differentiated Services Code Point) values tell routers how to prioritize packets. RingCentral uses industry-standard markings for voice traffic.

üîë Key DSCP Values for RingCentral

Voice/Audio (Highest Priority)

DSCP: EF (Expedited Forwarding)

46 (0x2E)

Used for RTP/SRTP voice packets

Signaling (High Priority)

DSCP: CS3 (Class Selector 3)

24 (0x18)

Used for SIP signaling packets

Video (High Priority)

DSCP: AF41 (Assured Forwarding)

34 (0x22)

Used for video conferencing

Default/Data (Normal Priority)

DSCP: CS0 (Best Effort)

0 (0x00)

Used for non-real-time traffic

üìù DSCP Priority Queue Recommendations

Configure your router/firewall with these priority queues:

  1. Queue 1 (Highest): DSCP EF (46) - Voice audio (RTP/SRTP) - 30% bandwidth guaranteed
  2. Queue 2 (High): DSCP AF41 (34) - Video - 25% bandwidth guaranteed
  3. Queue 3 (Medium): DSCP CS3 (24) - Signaling (SIP/TLS) - 10% bandwidth guaranteed
  4. Queue 4 (Normal): DSCP CS0 (0) - Best effort - Remaining bandwidth

‚öôÔ∏è Router Configuration for Desk Phones

Critical settings to check:

  1. Trust DSCP Markings: Router must respect DSCP values from phones (don't remark)
  2. Create Priority Queue: DSCP EF (46) ‚Üí Highest priority queue
  3. Bandwidth Allocation: Guarantee 30% for voice queue, 10% for signaling
  4. Port-Based QoS (if DSCP not available): Prioritize UDP 16384-16482 to highest queue
  5. Voice VLAN Tagging: If using voice VLAN, tag with CoS 5 (802.1p)

‚öôÔ∏è Desktop App QoS Configuration

Two-layer approach required:

1. Windows PC QoS Policy:
  • Open Group Policy Editor (gpedit.msc)
  • Navigate to: Computer Config ‚Üí Windows Settings ‚Üí Policy-based QoS
  • Create policy for RingCentral.exe
  • Set DSCP value to 46 for audio ports (UDP 10000-20000)
  • Set throttle rate (optional): Limit to prevent bandwidth hogging
2. Router QoS Configuration:
  • Trust DSCP markings from internal network
  • Create queue for DSCP EF (46) with highest priority
  • Allocate 30% bandwidth to voice queue
  • Alternative: Use port-based QoS for UDP 10000-20000
